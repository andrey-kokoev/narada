# @narada2/windows-komorebi-yasb-kit

Descriptor templates for Windows Komorebi/YASB configuration materialization.

This package models parameterized workspace/status-bar roles and health descriptors. It does not import user preference authority, live monitor content assignments, YASB vendor modifications, desktop-sunroom-2 runtime state, or logs.

## Materialization Policy

The `narada.windows_komorebi_yasb.materialization_request.v0` contract describes how a receiving PC Site would plan Komorebi/YASB template materialization through an operator-surface MCP path or an explicitly justified PC Site fallback. Decisions are descriptor-only and perform no restart, repair, file copy, or runtime projection mutation.

The live runtime directory is refused as authority. Imported live YASB runtime config, monitor state, user preference authority, and credentials are also refused.
