# Task 126: Correct Task 121 Safe Trial Coherence And Entry Ordering

## Why

Task 121 landed a valid `narada demo` surface, but the "safe local trial" path is still semantically incoherent.

Today, `narada init-repo --demo` creates a scope that still looks like a live Graph-backed mailbox operation, and `narada preflight demo` still hard-fails on live Graph credentials. That directly violates the intended contract of the safe local trial path:

- no live mailbox credentials required
- no live mailbox source required
- should still feel close to the real operational path

The first-run docs also still lead with the live mailbox gold path instead of the intended ordering:

1. show me
2. try safely
3. go live

## Findings Being Corrected

### 1. Safe trial still depends on Graph-shaped readiness

`packages/ops-kit/src/readiness/collect.ts` applies Graph credential checks globally, including demo/trial scopes.

That means the supposedly non-live trial path cannot become genuinely ready without live secrets.

### 2. Demo repo config is still modeled as a live Graph scope

`packages/ops-kit/src/commands/init-repo.ts` currently seeds `--demo` repos with:

- `sources: [{ type: "graph", user_id: "demo@example.com" }]`

That is the wrong substrate for a safe local trial repo. It teaches the wrong shape and forces fake failure messaging.

### 3. Docs still present the wrong entry order

`QUICKSTART.md` still starts with the real mailbox gold path and does not explicitly present:

- `narada demo` as the first "show me" path
- `narada init-repo --demo` as the first safe local trial path

## Goal

Make the non-live first-run surfaces fully coherent:

- `narada demo` is the zero-setup taste
- `narada init-repo --demo` is the near-real but still non-live trial repo path
- the real mailbox path comes after those two

## Required Outcomes

### 1. Make `--demo` truly non-live

The repo generated by:

```bash
narada init-repo --demo ~/src/my-tryout
```

must not require any live Graph credentials to be considered trial-ready.

That means:

- do not seed a live `graph` source into the demo scope
- use a fixture-backed, mock-backed, or explicitly local synthetic source shape
- ensure the resulting operation still looks like a real Narada operation, not a toy one-off

### 2. Fix readiness semantics for non-live trial scopes

`narada preflight demo` in a demo repo must not fail on Graph credentials when the scope is intentionally non-live.

The readiness model should distinguish between:

- live mailbox operations, which require Graph credentials
- non-live local trial operations, which do not

This must be done semantically, not by telling the user that failure is "expected."

### 3. Keep the trial path close to real operation shaping

The demo repo should still exercise the real user shape as much as possible:

- operation config exists
- operation can be inspected
- operation can be explained
- operation can be activated
- operation can be considered ready for its declared non-live mode

If any behavior is intentionally different from the live path, that difference must be explicit and explained.

### 4. Fix first-run documentation ordering

Update first-run docs so the entry paths are ordered as:

1. `narada demo`
2. `narada init-repo --demo ...`
3. real mailbox gold path

This applies at minimum to:

- `QUICKSTART.md`
- generated demo repo `README.md`
- any root README section that introduces how a new user should first try Narada

### 5. Explanatory output must stay honest

Remove messaging such as:

- "missing Graph credentials — expected for trial"

That is not a coherent product surface.

Instead, the output should explain:

- this operation is using a non-live source
- this trial path does not touch external systems
- what would change when switching to a live mailbox operation

## Deliverables

- corrected `init-repo --demo` scope generation
- corrected readiness logic for non-live trial scopes
- corrected generated demo repo README
- corrected top-level first-run docs ordering
- explanatory copy aligned with the actual trial mechanics

## Definition Of Done

- [ ] `narada init-repo --demo <path>` creates a repo whose starter operation is genuinely non-live
- [ ] `narada preflight demo` in that repo does not fail on absent Graph credentials
- [ ] the demo/trial repo still supports `inspect`, `explain`, and `activate` coherently
- [ ] first-run docs present demo first, safe trial second, live mailbox third
- [ ] no docs claim an intentional readiness failure as part of the safe trial UX

## Notes

This is not a request to weaken readiness for live operations.

Live mailbox operations must continue to hard-require Graph credentials.

The correction is about giving non-live operations a truthful operational shape instead of pretending a broken live shape is acceptable for trial.
