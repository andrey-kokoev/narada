### 20260516-1301-define-narada-native-provider-adapter-execution-contract (review-20260516-1301-define-narada-native-provider-adapter-execution-contract-1778890859105)

Verdict: **rejected**

- [x] **blocking**: Provider adapter registration is not wired into the Narada-native governed work path. runGovernedTaskHandoff always calls runFixtureWorkLoop, and runFixtureWorkLoop always calls invokeAdapter from the fixture adapter, so a configured provider_kind is readiness-visible but never selected during normal carrier task execution.
- [x] **blocking**: The shared provider runner is only exercised directly by tests and helper callers. No supervisor, work-loop, or task-handoff path reads the persisted provider registration and invokes executeProviderAdapter, so the execution contract does not yet make provider-backed intelligence usable by the carrier.
