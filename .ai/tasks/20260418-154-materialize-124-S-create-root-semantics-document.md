## Why

Semantics are currently split between `TERMINOLOGY.md`, audit artifacts, and scattered docs. That keeps ontology brittle.

## Goal

Create one root semantics/ontology document that consolidates the canonical user-facing and system-facing vocabulary.

## Deliverables

- single root `SEMANTICS.md` or `ONTOLOGY.md`
- consolidated definitions for core terms
- clear relationship to `TERMINOLOGY.md` and other docs

## Definition Of Done

- [ ] one root semantics document exists and is canonical
- [ ] overlapping docs point to it coherently
- [ ] core terms are defined in one place without contradiction
